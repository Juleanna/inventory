🔐 Налаштування безпеки

Зміна паролю адміністратора

Стандартний пароль securepassword123 необхідно змінити негайно!

  1. Увійдіть в адмін-панель з стандартними даними
  2. В правому верхньому куті натисніть на "admin"
  3. Оберіть "Change password"
  4. Введіть поточний пароль: securepassword123
  5. Встановіть новий надійний пароль
🔒 Вимоги до пароля
  • Мінімум 12 символів
  • Великі та малі літери (A-z)
  • Цифри (0-9)
  • Спеціальні символи (!@#$%^&*)
  • Не використовувати словникові слова
  • Не використовувати особисту інформацію

Приклад надійного пароля: Inv3nt0ry#2024$SeCuRe!

Створення безпечного секретного ключа

Django SECRET_KEY використовується для криптографічних операцій.

# Зупиніть систему
docker-compose down

# Згенеруйте новий секретний ключ
docker-compose run --rm web python -c "
import secrets
import string
alphabet = string.ascii_letters + string.digits + '!@#$%^&*(-_=+)'
secret_key = ''.join(secrets.choice(alphabet) for i in range(50))
print('SECRET_KEY=' + secret_key)
"

Тестування відправки email

Перевірте налаштування email через Django shell:

# Відкрийте Django shell
docker-compose exec web python manage.py shell

# В shell виконайте:
from django.core.mail import send_mail
from django.conf import settings

# Тест відправки
result = send_mail(
    subject='Тест email з Inventory System',
    message='Якщо ви отримали цей лист, налаштування email працюють правильно!',
    from_email=settings.DEFAULT_FROM_EMAIL,
    recipient_list=['your-email@example.com'],  # Вкажіть вашу пошту
    fail_silently=False,
)

print(f'Email надіслано: {result == 1}')
✅ Успішне тестування
Якщо команда повернула Email надіслано: True та ви отримали лист - налаштування правильні!

🗄️ Налаштування бази даних

Перевірка з'єднання з базою даних

Переконайтесь що PostgreSQL працює правильно:

# Перевірка статусу PostgreSQL
docker-compose exec db pg_isready -U postgres -d inventory_db

# Перевірка з'єднання з Django
docker-compose exec web python manage.py check --database default

# Перевірка кількості таблиць
docker-compose exec db psql -U postgres -d inventory_db -c "\dt"

# Інформація про базу даних
docker-compose exec db psql -U postgres -c "\l+ inventory_db"
✅ Успішний результат

Якщо команди виконались без помилок, база даних працює правильно:

  • pg_isready повертає "accepting connections"
  • check --database показує "System check identified no issues"
  • \dt показує список таблиць Django
Створення backup стратегії

Налаштуйте автоматичне резервне копіювання:

# Ручне створення backup
docker-compose exec db pg_dump -U postgres -v --clean --no-owner --format=custom inventory_db > backup_$(date +%Y%m%d_%H%M%S).dump

# Створення backup в SQL форматі
docker-compose exec db pg_dump -U postgres inventory_db > backup_$(date +%Y%m%d).sql

# Перевірка розміру backup
ls -lh backup_*.dump backup_*.sql
Автоматичний backup (Windows Task Scheduler):
💡 Поради щодо backup
  • Частота: щодня для активного використання
  • Зберігання: мінімум 30 днів
  • Тестування: щомісяця перевіряйте відновлення
  • Локація: зберігайте backup в іншому місці
Налаштування індексів для оптимізації

Створіть додаткові індекси для покращення продуктивності:

# Відкрийте PostgreSQL
docker-compose exec db psql -U postgres -d inventory_db

-- Індекси для швидкого пошуку обладнання
CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_equipment_serial_number 
ON inventory_equipment (serial_number);

CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_equipment_inventory_number 
ON inventory_equipment (inventory_number);

CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_equipment_name 
ON inventory_equipment USING gin(to_tsvector('english', name));

-- Індекс для фільтрації за категорією
CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_equipment_category 
ON inventory_equipment (category_id);

-- Індекс для фільтрації за локацією  
CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_equipment_location 
ON inventory_equipment (location_id);

-- Індекс для швидкого пошуку по датах
CREATE INDEX CONCURRENTLY IF NOT EXISTS idx_equipment_purchase_date 
ON inventory_equipment (purchase_date);

-- Перевірка створених індексів
\di+ inventory_equipment*
⚠️ Створення індексів
  • Використовуйте CONCURRENTLY щоб не блокувати таблицю
  • Індекси займають додатковий дисковий простір
  • Забагато індексів сповільнює операції INSERT/UPDATE
  • Моніторте використання індексів
Налаштування моніторингу БД 
# Перевірка активних з'єднань
docker-compose exec db psql -U postgres -c "
SELECT pid, usename, application_name, client_addr, state, query_start 
FROM pg_stat_activity 
WHERE datname = 'inventory_db';"

# Статистика використання таблиць
docker-compose exec db psql -U postgres -d inventory_db -c "
SELECT schemaname, tablename, n_tup_ins, n_tup_upd, n_tup_del, n_live_tup 
FROM pg_stat_user_tables;"

# Розмір бази даних
docker-compose exec db psql -U postgres -c "
SELECT pg_size_pretty(pg_database_size('inventory_db')) as database_size;"

Автоматичний backup бази даних 
# Щоденний backup (додайте в cron/task scheduler)
docker-compose exec db pg_dump -U postgres inventory_db > backup_$(date +%Y%m%d).sql

# Windows Task Scheduler
docker-compose exec db pg_dump -U postgres inventory_db > backup_%date%.sql
Відновлення з backup 
# Зупинити систему
docker-compose down

# Запустити тільки базу даних
docker-compose up -d db

# Відновити дані
docker-compose exec -T db psql -U postgres inventory_db < backup_20240101.sql

# Запустити систему
docker-compose up -d

🚨 Вирішення проблем

Типові проблеми та рішення

Проблема: Не можу увійти в систему
Можливі причини:
  • Неправильний логін/пароль
  • Контейнер не запущений
  • База даних недоступна
  • Проблеми з мережею

Рішення:

# 1. Перевірити статус контейнерів
docker-compose ps

# 2. Переглянути логи
docker-compose logs -f web

# 3. Перевірити підключення до БД
docker-compose exec web python manage.py check --database default

# 4. Скинути пароль адміністратора
docker-compose exec web python manage.py changepassword admin
Проблема: Повільна робота системи
Діагностика:
# Перевірити використання ресурсів
docker stats

# Перевірити розмір бази даних
docker-compose exec db psql -U postgres -c "\l+"

# Перевірити логи на помилки
docker-compose logs --tail=100 web

Оптимізація:

  • Збільшити пам'ять для Docker
  • Очистити старі логи
  • Оптимізувати індекси БД
  • Архівувати старе обладнання
Проблема: Не генеруються QR-коди 
# Перевірити встановлення бібліотек
docker-compose exec web python -c "import qrcode; print('QR OK')"
docker-compose exec web python -c "from barcode import Code128; print('Barcode OK')"

# Перевірити права на запис
docker-compose exec web ls -la /app/media/

# Переустановити залежності
docker-compose exec web pip install qrcode[pil] python-barcode
Проблема: Email не надсилаються

Перевірка налаштувань:

# Тестування email в Django shell
docker-compose exec web python manage.py shell

# В shell виконати:
from django.core.mail import send_mail
send_mail(
    'Test Subject',
    'Test message',
    'from@company.com',
    ['to@company.com'],
    fail_silently=False,
)

Типові помилки:

  • Неправильні SMTP налаштування
  • Блокування портів провайдером
  • Неправильний пароль додатку (Gmail)
  • Відсутність SSL/TLS
