1. Передумови

Перед початком переконайтесь, що на вашій машині встановлено наступне програмне забезпечення:

Програма Мінімальна версія Перевірка
Docker 20.10+ docker --version
Docker Compose 2.0+ docker compose version
Git 2.30+ git --version
💡 Примітка для Windows Рекомендовано використовувати Docker Desktop для Windows з WSL 2 бекендом. Переконайтесь, що Docker Desktop запущений перед виконанням команд.

Встановлення Docker (якщо не встановлено)

Windows:

# Завантажте та встановіть Docker Desktop з офіційного сайту:
# https://docs.docker.com/desktop/install/windows-install/
# Після встановлення перезавантажте комп'ютер

Linux (Ubuntu/Debian):

sudo apt update
sudo apt install -y docker.io docker-compose-plugin
sudo usermod -aG docker $USER
# Перезайдіть в систему після цієї команди

3. Порти та мережа

Оскільки порти 80, 8080 та 6379 вже зайняті на вашій машині, проект налаштований на альтернативні порти:

Сервіс Порт на хості Порт у контейнері Змінна в .env
Nginx (додаток) 8081 80 APP_PORT
PostgreSQL 5433 5432 DB_PORT_EXTERNAL
Redis 6380 6379 REDIS_PORT_EXTERNAL
⚠️ Важливо Якщо ці порти також зайняті, змініть їх у файлі .env. Наприклад, APP_PORT=9090 запустить додаток на http://localhost:9090.

4. Налаштування змінних оточення (.env)

1

Скопіюйте файл прикладу

# Windows (PowerShell)
Copy-Item .env.example .env

# Windows (CMD)
copy .env.example .env

# Linux / macOS / Git Bash
cp .env.example .env
2

Відредагуйте .env файл

Відкрийте .env у текстовому редакторі та змініть наступні значення:

# ОБОВ'ЯЗКОВО змініть ці значення:

# Генеруйте унікальний ключ (мін. 50 символів)
SECRET_KEY=ваш-унікальний-секретний-ключ-тут

# Надійний пароль для бази даних
DB_PASSWORD=ваш-надійний-пароль-бд

# Для production: вимкніть DEBUG
DEBUG=True

# Порти (змініть якщо потрібно)
APP_PORT=8081
DB_PORT_EXTERNAL=5433
REDIS_PORT_EXTERNAL=6380

# Суперкористувач (буде створений автоматично)
DJANGO_SUPERUSER_USERNAME=admin
DJANGO_SUPERUSER_EMAIL=admin@example.com
DJANGO_SUPERUSER_PASSWORD=ваш-пароль-адміна
3

Згенеруйте SECRET_KEY (опціонально)

# Використайте Python для генерації безпечного ключа:
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"

# Або без Django:
python -c "import secrets; print(secrets.token_urlsafe(50))"
🔒 Безпека Файл .env містить паролі та секретні ключі. НІКОЛИ не додавайте його у Git-репозиторій. Він вже є в .gitignore.

5. Перший запуск

1

Перейдіть у директорію проекту

cd d:\inventory
# або для Linux/macOS:
cd /path/to/inventory
2

Зберіть Docker-образи

Ця команда завантажить базові образи та зберё проект. Перший раз це може зайняти 5-10 хвилин.

docker compose build

Очікуваний вивід:

[+] Building 120.5s (25/25) FINISHED
 => [web] ...
 => [frontend] ...
 => exporting layers
3

Запустіть усі сервіси

# Запуск у фоновому режимі
docker compose up -d

Очікуваний вивід:

[+] Running 7/7
 ✔ Container inventory_db       Started
 ✔ Container inventory_redis    Started
 ✔ Container inventory_web      Started

4

Дочекайтесь ініціалізації

Перший запуск виконує міграції бази даних та збирає статичні файли. Зачекайте 30-60 секунд, потім перевірте стан:

# Перевірка стану контейнерів
docker compose ps

7. Корисні команди

Управління контейнерами

# Запустити всі сервіси
docker compose up -d

# Зупинити всі сервіси
docker compose stop

# Перезапустити конкретний сервіс
docker compose restart web

# Перезібрати та перезапустити (після змін коду)
docker compose up -d --build

# Перезібрати тільки frontend
docker compose build frontend
docker compose up -d frontend nginx

Django management команди

# Виконати міграції вручну
docker compose exec web python inventory_project/manage.py migrate

# Створити нового суперкористувача
docker compose exec web python inventory_project/manage.py createsuperuser

# Зібрати статичні файли
docker compose exec web python inventory_project/manage.py collectstatic --noinput

# Django shell
docker compose exec web python inventory_project/manage.py shell

# Створити нову міграцію
docker compose exec web python inventory_project/manage.py makemigrations

Робота з базою даних

# Підключитися до PostgreSQL
docker compose exec db psql -U postgres -d inventory_db

# Або з хост-машини (порт 5433)
psql -h localhost -p 5433 -U postgres -d inventory_db

# Підключитися до Redis
docker compose exec redis redis-cli

8. Перегляд логів

# Всі логи (останні 100 рядків)
docker compose logs --tail=100

# Логи конкретного сервісу в реальному часі
docker compose logs -f web
docker compose logs -f celery_worker
docker compose logs -f nginx
docker compose logs -f frontend

# Логи кількох сервісів одночасно
docker compose logs -f web celery_worker

# Логи Django з файлу (всередині контейнера)
docker compose exec web cat /app/logs/inventory.log
docker compose exec web tail -f /app/logs/inventory.log

9. Оновлення проекту

Коли ви внесли зміни в код або отримали оновлення з Git:

1

Отримайте останні зміни

git pull origin main
2

Перезберіть та перезапустіть

# Перезібрати образи та перезапустити контейнери
docker compose up -d --build

Docker автоматично виконає:

  • Перезбірку backend і frontend образів
  • Міграції бази даних (через entrypoint.sh)
  • Збір статичних файлів
  • Перезапуск усіх сервісів
3

Якщо змінилися залежності

# Повна перезбірка без кешу (якщо змінились requirements.txt або package.json)
docker compose build --no-cache
docker compose up -d

10. Резервне копіювання

Бекап бази даних

# Створити бекап PostgreSQL
docker compose exec db pg_dump -U postgres -d inventory_db > backup_$(date +%Y%m%d_%H%M%S).sql

# Windows PowerShell варіант:
docker compose exec db pg_dump -U postgres -d inventory_db > "backup_$(Get-Date -Format 'yyyyMMdd_HHmmss').sql"

Відновлення з бекапу

# Відновити базу з файлу бекапу
cat backup_20260316.sql | docker compose exec -T db psql -U postgres -d inventory_db

# Windows:
Get-Content backup_20260316.sql | docker compose exec -T db psql -U postgres -d inventory_db

Бекап медіа-файлів

# Скопіювати медіа з Docker volume
docker compose cp web:/app/media ./backup_media

11. Вирішення проблем

Проблема: Контейнер не запускається

Діагностика

# Подивіться логи проблемного контейнера
docker compose logs web
docker compose logs db

# Перевірте стан контейнерів
docker compose ps -a

Проблема: Порт вже зайнятий

Помилка: bind: address already in use 
# 1. Знайдіть процес, що займає порт
# Windows:
netstat -ano | findstr :8081

# Linux:
sudo lsof -i :8081

# 2. Або змініть порт у .env:
APP_PORT=9090

# 3. Перезапустіть
docker compose up -d

Проблема: Помилка підключення до БД

# Перевірте, що контейнер db запущений та здоровий
docker compose ps db

# Перевірте логи БД
docker compose logs db

# Перезапустіть БД та web
docker compose restart db web

Проблема: Frontend показує білу сторінку

# 1. Перевірте логи збірки frontend
docker compose logs frontend

# 2. Перевірте, що nginx проксує правильно
docker compose logs nginx

# 3. Перезберіть frontend
docker compose build --no-cache frontend
docker compose up -d frontend nginx

Проблема: Помилки міграцій

# Запустіть міграції вручну та подивіться на помилку
docker compose exec web python inventory_project/manage.py migrate --verbosity=2

# Якщо потрібно скинути міграції (УВАГА: видалить всі дані!)
docker compose exec web python inventory_project/manage.py flush --noinput
docker compose exec web python inventory_project/manage.py migrate

Проблема: Немає місця на диску

# Очистити невикористані Docker ресурси
docker system prune -a --volumes

# Перевірити використання диску Docker
docker system df

Повне скидання (крайній випадок)

⛔ УВАГА: Ця дія видалить ВСІ дані! Усі дані в базі, Redis, медіа-файли — будуть втрачені назавжди. 
# Зупинити і видалити все (контейнери, мережі, томи)
docker compose down -v --remove-orphans

# Перезібрати з нуля
docker compose build --no-cache
docker compose up -d

12. Зупинка та видалення

Тимчасова зупинка (дані збережуться)

# Зупинити всі контейнери (дані залишаться у volumes)
docker compose stop

# Запустити знову
docker compose start

Видалення контейнерів (дані збережуться)

# Видалити контейнери та мережі, але зберегти volumes (дані)
docker compose down

Повне видалення (з даними)

# Видалити все, включаючи дані (volumes)
docker compose down -v

# Видалити також зібрані образи
docker compose down -v --rmi all
